Overview

Out here, “no service” is normal. What matters is whether someone else knows your plan, your boundaries, and when to worry.

RidgeRelay is a concept for a lightweight safety system built around planning: intent, timing, and check-ins. If check-ins stop, escalation follows clear steps — starting with your contacts and only moving toward emergency support when needed.

Priorities

  • Planning over posting
  • Clarity over complexity
  • Trust through predictable steps
  • Static + auditable (prototype)

How it works

A simple 4-step flow — built to stay understandable under stress.

  • 🧭
    Plan Declare intent, start time, return time, and emergency contact.
  • 🧱
    Guardrails Set boundaries so someone else understands the plan quickly.
  • ⏱️
    Check-ins Choose a cadence that matches conditions (30–90 minutes).
  • 🚨
    Escalation Progressive steps if you miss check-ins — no surprises, just clarity.

FAQ

Short answers, clear expectations.

Does RidgeRelay track me?
No. This prototype is a static GitHub Pages site. It doesn’t create accounts and doesn’t send data anywhere.
What happens if I lose service?
The concept focuses on what you can set before you lose signal: intent, timing, check-ins, and escalation expectations. Offline features are future scope.
How is this different from AllTrails or Google Maps?
Those tools help you navigate. RidgeRelay focuses on safety planning and escalation clarity: who’s notified, when, and with what information.
Is the demo trail line accurate?
It’s illustrative only. The GeoJSON line exists to demonstrate loading a local trail overlay on the Leaflet map.
What is “lost pet seen on trail” for?
It’s an optional utility: record details and copy/share them quickly. In this demo it saves locally only (no uploads, no feed).
Can I save my plan?
Yes (demo): “Save plan (Local)” uses your browser’s localStorage. It’s stored on your device only.

About

RidgeRelay is a concept / school project prototype focused on safety-first planning UX.

This site demonstrates a calm planning flow using static hosting and open-source tools (Leaflet + OpenStreetMap). The demo is intentionally scoped to a single trail to keep it clear, testable, and submission-ready.

Prototype scope

  • Single-page concept
  • Planner module + map
  • Local demo GeoJSON overlay
  • Local-only save/copy tools
  • No backend, no accounts
