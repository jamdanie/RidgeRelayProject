About RidgeRelay

RidgeRelay is a conceptual outdoor safety planning interface designed for environments where cellular connectivity may be limited or unavailable.

Project Purpose

Most outdoor safety tools rely heavily on continuous GPS tracking, reliable cellular networks, or emergency alert systems that activate only after a problem has occurred.

RidgeRelay explores a different idea: declaring intent before going offline.

By encouraging hikers, runners, and adventurers to define their trip expectations ahead of time, the system focuses on preventative safety awareness instead of reactive alerts.

What This Prototype Demonstrates

  • Interactive trail explorer with filtering
  • Trail detail drawer with photos and trail information
  • Route preview map using Leaflet and GeoJSON
  • Weather preview loaded from local JSON demo data
  • Mobile-first responsive layout
  • LocalStorage wishlist functionality

These features demonstrate interface concepts without requiring a backend server.

Design Principles

The RidgeRelay interface was designed using several Human-Computer Interaction (HCI) concepts.

Intent Before Interface

Users declare trip expectations before entering low-connectivity areas. This helps reduce uncertainty and supports shared situational awareness.

Progressive Disclosure

Important details appear only when needed. Trail cards show basic information while the drawer reveals maps, photos, and weather data.

Calm Interaction Design

The interface intentionally avoids alarm-style visual patterns. Instead it uses clean typography, restrained color accents, and clear hierarchy to support calm decision-making outdoors.

Prototype Limitations

This repository represents a frontend demonstration only.

  • No backend infrastructure
  • No real GPS tracking
  • No emergency dispatch services
  • No user accounts or cloud storage
  • Weather data is simulated using local JSON

These constraints allow the project to be deployed entirely as a static website using GitHub Pages.
